1、真实角色：被代理的角色(委托类)--XiaoMin.java
2、抽象角色：一般使用接口或者抽象类来实现--Person.java
3、代理角色：代理真实角色，代理真实角色后一般会做一些附属的操作--MeiPo.java

代理类里步奏：
1、实现InvocationHandler接口
2、私有的Object的代理对象
3、建立代理对象和真实对象的代理关系，并返回代理对象（bind绑定）
4、重写invoke方法


Q：jdk动态代理为什么一定要实现一个接口呢？
A：因为生成代理对象时需要通过反射接口来获取所有方法列表。为啥要通过接口来获取而不是直接通过被代理类来获取呢？
因为Proxy.newProxyInstance()方法生成的代理对象$Proxy0.class 是 继承了 Proxy 类，由于只能单继承所以只能通过实现接口来反射获取
被代理对象的方法列表了。
